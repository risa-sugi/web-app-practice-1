# プロジェクト名

## 概要
このリポジトリは、個人開発でWebアプリを企画・開発・公開・運用するためのプロジェクト管理用リポジトリです。  
現時点ではコード未着手で、要件整理、技術選定、設計メモ、開発ナレッジの蓄積を主な目的としています。

## 目的
- 個人で公開まで進められるWebアプリを作る
- MVPを素早く定義して、早い段階でリリースする
- 壁打ち内容、意思決定、実装時の学びをナレッジとして蓄積する

## 解決したい課題
- ここに「誰の、どんな課題を解決したいか」を書く
- まだ決まっていなければ仮置きでもOK

例:
- 〇〇な人が、△△に時間や手間がかかっている課題を解決する
- ××をもっと簡単・直感的にできるようにする

## 想定ユーザー
- ここに想定ユーザーを書く
- まだ広くてもよいので、まずは仮説を書く

例:
- 個人で学習管理をしたい人
- 日々の記録をシンプルに残したい人
- 情報を整理したいが複雑なツールは使いたくない人

## 提供したい価値
- ユーザーがこのアプリを使うことで何が良くなるか
- どんな体験を提供したいか

例:
- 数分で目的を達成できる
- 面倒な作業を減らせる
- 継続しやすくなる
- 見やすく、迷わず使える

## MVPで実装したいこと
- 最初の公開までに必要な最小機能を書く
- 思いついた機能を全部入れず、最小限に絞る

例:
- ユーザー登録 / ログイン
- データの作成
- 一覧表示
- 編集 / 削除
- 最低限の検索や絞り込み

## 今はやらないこと
- 後回しにする機能を書く
- スコープを守るために重要

例:
- 通知機能
- SNS連携
- 高度な分析機能
- 管理画面
- ネイティブアプリ対応

## 開発方針
- MVPを優先し、過剰設計を避ける
- 個人で保守しやすい構成を優先する
- 実装速度、保守性、運用コストのバランスを重視する
- 曖昧な要件は推測で決めず、判断に重要なものは確認して整理する
- 壁打ち内容や学びは、再利用できる形でナレッジ化する
