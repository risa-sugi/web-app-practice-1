# web-app-practice-1
天気×AIコーディネート提案アプリ — 今日の天気とシーンに合ったコーデをAIが提案

## 概要
天気情報とシーン（通勤・デート・カジュアル等）に応じて、AIがおすすめの服装・カラーを提案するWebアプリ。
参考コーデ画像も表示し、毎朝の服選びをサポートする。

## 目的
- 個人でWebアプリの企画〜開発〜公開〜運用を一通り経験する
- AI / 外部API連携の実装スキルを身につける
- MVPを1ヶ月でリリースする

## 解決したい課題
- 毎朝の天気に合った服選びに迷う
- 気温や天候を見ても、具体的に何を着ればいいかわからない
- シーンに合ったコーディネートを手軽に知りたい

## 想定ユーザー
- 毎朝の服選びに悩む人
- 天気に合った服装を手軽に知りたい人
- コーディネートのバリエーションを増やしたい人

## 提供したい価値
- 天気を確認するついでに、コーデ提案が得られる
- シーンに合った具体的な服装がわかる
- おすすめカラーで色選びの参考になる
- 参考画像で完成イメージがつかめる

## MVPで実装すること
- ユーザー登録 / ログイン
- 天気情報の取得・表示（位置情報 or 地域手動選択）
- シーン選択（通勤 / デート / カジュアル / アウトドア / フォーマル）
- AIコーデ提案（テキスト + おすすめカラー）
- 参考コーデ画像の表示（Unsplash API）
- コーデの再生成（別パターン）
- 地域設定

## 今はやらないこと
- パーソナルカラー診断・設定（v2最有力候補）
- お気に入りコーデの保存・履歴
- 自分の服を登録してその中から提案
- 週間天気対応
- SNSシェア機能

## 技術スタック
- Frontend + API: Next.js（App Router / TypeScript）
- Database: Supabase PostgreSQL
- Auth: Supabase Auth
- UI: shadcn/ui + Tailwind CSS
- Weather API: OpenWeatherMap One Call API 3.0
- AI: Google Gemini 2.5 Flash
- Images: Unsplash API
- Hosting / Deploy: Vercel

## 開発方針
- MVPを優先し、過剰設計を避ける
- 個人で保守しやすい構成を優先する
- 実装速度、保守性、運用コストのバランスを重視する
- 壁打ち内容や学びは、再利用できる形でナレッジ化する
- 全サービスを無料枠内で運用する
